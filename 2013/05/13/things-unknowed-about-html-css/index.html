你可能不知道的HTML和CSS的一些事情

发表于 2013-05-13 | 分类于 html,css

我最近在《Fronteers　2012》看了一个很好的马蒂亚斯的视频演示。我觉得既有趣又励志。我心中已经有了一些类似的事情，所以我沉静下来写这篇文章。这是关于一些随机的东西和事实，你可能或者还没有了解的HTML和CSS。

CSS em单位

原来，CSS里的em单位是来自width的大写字母M，这就是它名字的出处。我想起了我很久之前看的一本关于CSS的书，不过我已经忘记那本书的名字，不过你可以去维基百科查阅更多这个话题的知识。

HTML5标签

HTML5结构元素是当时在广泛的CSS类和id选择器之间命名搜索中发现的一个实际的实施。我通过由布鲁斯·劳森和雷米夏普写的《HTML5 的介绍》第一次读到目前的HTML5标签如何产生的。

CSS特异性

它需要256个链接的CSS类选择符去打败一个id的特异性。当Chris Coyier做这个例子的时候，已经有很多关于这个主题的讨论。瑞安塞登也写了一些想法。
此外，在这个话题上，发现一个有趣的事实是：在css里对CSS类名和ID名没有最大长度的限制。

颜色

秘鲁番茄实际上是有效的CSS颜色名称。同时，十六进制#bada55的颜色不那么糟糕，它实际上是绿黄色。
此外，当HTML bgcolor属性（不建议这样子使用）：<body bgcolor="chucknorris"></body>时，一片血腥的红色就会呈现。 马蒂亚斯在他的视频提到查克·诺里斯的敌人的血。也许是纯属巧合，也许不是。
阅读 Stack Overflow的讨论，找出为什么HTML认为chucknorris是一种颜色

不再有css4

似乎终究是没有CSS4这样的东西的了。同时，CSS3是继CSS2之后的一切，包括它的所有模块。

HTTP 请求

“最快的HTTP请求，一个都没有 。”
这与HTML &CSS无关，但我喜欢很喜欢这句话。每次我做一个项目，并尝试优化HTTP请求数的时候，这句话就会从我的脑海里弹出。我想我第一次看到它，是阅读Steve Souders 的邮件。

::before &； ::after 伪元素

直到前段时间，火狐是唯一一个支持CSS伪元素，如::before和::after的过渡和动画。这是一个很大很重要的角色，并且现在也得到了Webkit的支持，这是2013年的一个好消息。现在，您可以打开Google Chrome，就能看到他们在活动。
