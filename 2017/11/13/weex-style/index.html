需要关注的一些 weex 样式

发表于 2017-11-13 | 分类于 weex

需要关注的一些 weex 样式

  1. Weex 盒模型基于 CSS 盒模型 。box-sizing 默认为 border-box，即盒子的宽高包含内容、内边距和边框的宽度，不包含外边距的宽度。
  2. Weex 对于长度值目前只支持 px 长度单位，不支持相对单位（em、rem、pt），暂时不支持百分值％;
  3. weex 的样式不支持简写，譬如：border 目前不支持类似这样border: 1 solid #ff0000;的组合写法。
  4. Weex 目前不支持 z-index 设置元素层级关系，但靠后的元素层级更高，因此，对于层级高的元素，可将其排列在后面。
  1. 如果定位元素超过容器边界，在 Android 下，超出部分将不可见，原因在于 Android 端元素 overflow 默认值为 hidden，但目前 Android 暂不支持设置 overflow: visible。
  2. background-image 优先级高于 background-color，同时设置 background-image 和 background-color，background-color 被覆盖
  3. weex支持线性渐变linear-gradient，不支持径向渐变radial-gradient
  4. weex中box-shadow仅仅支持ios
  5. 目前<image>组件无法定义一个或几个角的border-radius，对ios有效，对android无效。直接使用border-radius，不用border-top-left-radius

其他的注意点：

1、weex在切换样式的时候不会重置所有样式，所以，切换前后的两个类都要设置属性值

2、图片外套边框的时候，父级不指定width和height值，避免部分图片溢出或者不贴边

最后

可以按照以下步骤来规划 Weex 页面的样式。

  • 全局样式规划：将整个页面分割成合适的模块。
  • flex 布局：排列和对齐页面模块。
  • 定位盒子：定位并设置偏移量。
  • 细节样式处理：增加特定的具体样式。
